# BISECITON

BISECTION is a unified, biologically informed deep learning framework designed for accurate unsupervised clustering and multi-scale biological interpretation of both single-cell RNA sequencing (scRNA-seq) and spatial transcriptomics (ST) data. Unlike traditional methods that focus solely on latent representation optimization while leaving downstream interpretation decoupled and post-hoc, BISECTION introduces an innovative architecture that harmonizes robust feature learning with intrinsic biological accountability. By coupling an attention-enhanced multi-neighbor contrastive learning module with a knowledge-guided teacher-student distillation network, the framework seamlessly maps statistical mathematical embeddings onto transparent, curated gene-set pathways. Consequently, BISECTION delivers competitive or superior clustering assignments while simultaneously providing highly accurate, trusted type and functional annotations at both the discrete cluster and single-cell/spot resolution.

## Installation

Create a Python environment and install the dependencies:

```bash
pip install -r requirements.txt
```

The workflow was tested with Python 3.8, PyTorch 1.13.1, and CUDA 11.3.

For GPU runs, install the PyTorch build that matches your CUDA version before running the workflow.

## Data

The Pancreas example data and MSigDB-derived gene-set files are included under `data/`. The configuration file uses relative paths:

```yaml
root_dir: ./data/datasets
gene_sets_path: ./data/gene_sets/human_function3
model_path: ./model/Pancreas
```

Large data files are tracked through Git LFS. If cloning from a remote repository, install Git LFS first:

```bash
git lfs install
git lfs pull
```

The repository license applies to the source code. Dataset and gene-set reuse should follow the original Pancreas dataset and MSigDB terms of use; see [DATA_AVAILABILITY.md](DATA_AVAILABILITY.md).

## Reproducing the Pancreas Example

The recommended entry point is the notebook:

```text
examples/Pancreas.ipynb
```

Open the repository root as the working directory, start Jupyter or VS Code, and run the notebook from top to bottom. The first notebook cell automatically resolves the project root when opened from `examples/`.

Run all commands from the repository root so generated validation summaries, figures, checkpoints, and embeddings are written to the expected project-relative output directories.

The script entry point can also be used:

```bash
python main.py --device auto
```

Generated checkpoints, figures, validation summaries, and embeddings are ignored by Git.

## Reproducibility Notes

- The original model architecture, loss functions, training loops, and hyperparameters are preserved.
- Gene-set files are loaded with a deterministic sorted order.
- The notebook keeps representative figure outputs for preview, while generated checkpoints, validation summaries, and embeddings can be regenerated by the user.
- Machine-specific absolute output paths were replaced with repository-relative paths.
